## Cronologia dos problemas

### 1. PM2 + venv: 2h de debugging
O webhook-whatsapp falhava com `ModuleNotFoundError: No module named 'fastapi'` apesar de:
- `--interpreter .venv/bin/python3` estar correto
- `python3 -c "import fastapi"` funcionar no terminal interativo
- `.venv/bin/activate` estar íntegro

**Causa raiz:** PM2 invoca o binário python3 do venv mas NÃO seta `VIRTUAL_ENV` no ambiente.
Sem essa variável, o Python importa do sistema, não do venv.

**Solução:** wrapper bash com `. /path/.venv/bin/activate && exec python3 script.py`,
iniciado com `pm2 start wrapper.sh --interpreter /bin/bash`.

### 2. Cloudflare: rotas fantasmas no YAML local
Ao tentar adicionar `sia.oesteodontologia.com.br` no config.yml local para "documentar",
o cloudflared quebrou com `mapping key "service" already defined at line 29`.

**Causa:** A rota `sia.oesteodontologia.com.br → localhost:8501` já existia no dashboard
do Cloudflare. O túnel baixa a config remota e faz merge — chaves duplicadas quebram o YAML.

**Correção:** remover a entrada duplicada e restart. Manter o config.yml local apenas
com as rotas que NÃO estão no dashboard (ex: `http_status:404`).

### 3. SSH config: linha órfã global
Após editar `~/.ssh/config` do Mac com `sed`, a linha `Host ssh.oesteodontologia.com.br`
foi substituída por `###` mas as diretivas indentadas seguintes (`User {{COMMANDER}}`, `ProxyCommand`)
ficaram como configurações GLOBAIS. Isso fez TODAS as conexões SSH autenticarem como `{{COMMANDER}}`.

**Sintoma:** `{{OVH_SSH_COMMAND}}` autenticava como `{{COMMANDER}}` mesmo com `User {{COMMANDER}}fae` no Host block.

**Correção:** comentar todas as linhas órfãs, não apenas o `Host`.

### 4. Evolution API: instâncias somem após re-ativação
Após ativar a licença no novo servidor, `fetchInstances` retornava `[]`. O banco PostgreSQL
foi migrado corretamente, mas as instâncias são atreladas ao `instance_id` antigo (derivado
do hardware/SO). É necessário recriar a instância Baileys via Manager UI.

### 5. SIA: uv sync não instala se não houver .venv
`uv sync` com o lock file presente mas sem `.venv` apenas verifica checksums — não instala.
É preciso `uv venv --python 3.12` primeiro, depois `uv sync` ou `uv pip install streamlit`.

### 6. Dontus Streamlit: startup lento
O container `dontus-app` demorou ~2 minutos para responder após `docker compose up -d`.
Durante esse período, `curl http://localhost:8502/` retornava `000` (conexão recusada).
**Mitigação:** aguardar 30s+ após `up -d` antes de testar.
