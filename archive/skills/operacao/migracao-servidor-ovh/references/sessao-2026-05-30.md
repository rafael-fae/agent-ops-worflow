## Erros encontrados e resolvidos

### 1. OpenCode CLI — npm não basta
- O pacote npm `@opencode-ai/plugin` gera um wrapper JS de 2.6 MB que dá Bus Error ao rodar
- O binário real é um ELF de 138 MB em `{{COMMANDER_HOME}}/.opencode/bin/opencode`
- Transferido via rsync OVH→OVH (rsync -avz, 30 MB/s, 4 segundos)

### 2. Venv com paths do servidor antigo
- `.venv/bin/activate` tinha `VIRTUAL_ENV='{{COMMANDER_HOME}}/...'` hardcoded
- `source activate` setava a variável para path inexistente → `ModuleNotFoundError`
- Afetou: webhook-whatsapp, webhook-meta, SIA
- Solução: `rm -rf .venv && uv venv --python 3.12 && uv sync`

### 3. PM2 não ativa venv mesmo com --interpreter
- `pm2 start script.py --interpreter .venv/bin/python3` invoca o binário mas não seta VIRTUAL_ENV
- Logs mostravam `ModuleNotFoundError: No module named 'fastapi'` mesmo com pacote instalado
- Solução: wrapper script bash + `--interpreter /bin/bash`
  ```bash
  #!/bin/bash
  . /path/.venv/bin/activate
  exec python3 /path/script.py
  ```

### 4. SSH config — linhas órfãs
- Ao editar `~/.ssh/config`, linhas `User {{COMMANDER}}` e `ProxyCommand` ficaram sem Host block
- Viraram configurações GLOBAIS, afetando todas as conexões
- `ssh ssh.oesteodontologia.com.br` autenticava como `{{COMMANDER}}` em vez de `{{COMMANDER}}fae`

### 5. Cloudflared YAML duplicate key
- Adicionar rota `sia.oesteodontologia.com.br` no config.yml local quando já existia remotamente
- Erro: `mapping key "service" already defined at line 32`
- Cloudflared não iniciava. A rota sia já estava no dashboard (localhost:8501).

### 6. HERMES_HOME quebrado no systemd do Lirin
- `Environment="HERMES_HOME={{COMMANDER_HOME}}fae/Dev/hermes-profiles/lirin"` no serviço
- Hermes não reconhecia o perfil: `Error: Profile 'lirin' does not exist`
- Solução: remover HERMES_HOME, usar symlink `~/.hermes/profiles/lirin → Dev/hermes-profiles/lirin`

### 7. Docker roshar-agents build pesado
- Imagem de 8.75 GB, build ~10 minutos
- Container separado do compose principal, em `{{COMMANDER_HOME}}/hermes-roshar/docker-compose.yml`
- Usa `network_mode: host`
- Precisa do diretório `/root/projects/cerebro_grupo` no host
